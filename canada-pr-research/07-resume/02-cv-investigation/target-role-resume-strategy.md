# Target Role Resume Strategy

## Purpose
This note keeps the resume aligned to the most realistic Canada job paths rather than becoming a generic document.

## Current Candidate Targets
- IT Support
- QA Tester
- Application Support
- Operations Support
- Junior Developer

## Strategy
- Build one base resume centered on transferable technical support and delivery strengths
- Tailor the summary, key skills, and top bullets for each job family later
- Start with the role that offers the best mix of realism, lower communication burden, and hiring volume

## Angle By Role
### IT Support
- Emphasize troubleshooting, user support, ticket handling, systems familiarity, and reliability

### QA Tester
- Emphasize test design, bug reporting quality, detail orientation, regression testing, and collaboration with developers

### Application Support
- Emphasize incident handling, root-cause investigation, production support, stakeholder coordination, and process discipline

### Operations Support
- Emphasize monitoring, routine execution, issue escalation, documentation, and operational stability

### Junior Developer
- Emphasize shipped work, code ownership, debugging, delivery, and practical tool usage

## Decision Rule
- If the current resume is weak for software development roles, do not force a Junior Developer-first strategy.
- If support and operations work is more provable, use that as the first Canada entry angle.
