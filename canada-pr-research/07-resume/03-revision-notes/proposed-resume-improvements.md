# Proposed Resume Improvements

## Purpose
This note is the first-pass proposal for how to improve the resume before tailoring it to Canada job applications.

## Recommended Revision Priorities
1. Clarify the target role before rewriting everything.
2. Replace generic responsibility bullets with evidence-based achievement bullets.
3. Tighten the skills section so it matches the target jobs.
4. Improve readability for recruiters who scan quickly.
5. Remove weak or distracting details that do not help the Canada application goal.

## Suggested Changes
### 1. Stronger top section
- Add a short professional summary only if it can clearly state your role direction, years of relevant experience, and strongest technical themes.
- Example direction:
  - Technical support / QA / application support professional with hands-on troubleshooting, cross-team coordination, and practical software delivery exposure.
- Avoid empty lines like "hardworking team player" unless backed by concrete evidence elsewhere.

### 2. Better bullet writing
- Rewrite bullets using a simple pattern:
  - Action + scope + tools + outcome
- Prefer:
  - Investigated production issues across X systems and reduced repeat incidents by improving handoff documentation.
- Avoid:
  - Responsible for handling issues and supporting the team.

### 3. Reduce unfocused content
- Remove old details that do not support the intended role.
- Compress minor technologies into grouped categories rather than long lists.
- Keep only projects that show skills a Canada employer would care about.

### 4. Make communication look stronger on paper
- Use direct, simple, professional English.
- Prefer shorter bullets with clear verbs.
- Cut wording that feels uncertain, apologetic, or overly internal.

### 5. Show practical value
- Highlight reliability, problem solving, cross-functional support, documentation, and execution discipline.
- These traits may help more than trying to look like a senior specialist if the evidence is not there.

## What I Will Look For When Revising Your Resume
- Unclear or weak job titles
- Bullets with no outcome or no scope
- Missing metrics that can realistically be added
- Skill lists that are too broad
- Experience that should be reordered, merged, or de-emphasized
- Projects that can strengthen the Canada-targeted story

## First Revision Pass Checklist
- [ ] Choose first target role
- [ ] Paste current resume text into a working note
- [ ] Mark every bullet as strong, weak, or unclear
- [ ] Rewrite summary and top 5-8 bullets first
- [ ] Recheck whether the final resume tells one coherent story
