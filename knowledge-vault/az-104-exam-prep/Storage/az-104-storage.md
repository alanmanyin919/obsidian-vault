# AZ-104 Storage

## Core Concepts
- Storage accounts and performance tiers
- Blob storage, file shares, queues, and tables
- Hot, cool, and archive access patterns
- Replication options such as locally redundant and geo-redundant designs

## Know How To Do
- Create storage accounts with appropriate redundancy and performance settings
- Configure blob containers and Azure Files shares
- Secure access with RBAC, keys, shared access signatures, and network restrictions
- Enable data protection features where appropriate

## Exam Traps
- Picking redundancy based only on cost
- Confusing account-level access with object-level permissions
- Ignoring public access and firewall exposure
- Forgetting differences between blob storage and Azure Files use cases

## Practice Tasks
- Create a storage account for general workloads
- Upload blobs and configure container access behavior
- Create a file share and mount it from a VM
- Restrict storage access by selected networks only
