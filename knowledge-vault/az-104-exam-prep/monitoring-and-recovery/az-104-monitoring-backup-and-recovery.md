# AZ-104 Monitoring, Backup, and Recovery

## Core Concepts
- Azure Monitor, metrics, logs, alerts, and action groups
- Log Analytics workspaces and operational visibility
- Azure Backup and recovery planning

## Know How To Do
- Create alerts for platform metrics and operational conditions
- Route diagnostics and logs to the right destination
- Use Log Analytics queries for investigation
- Configure backup for supported workloads and validate restore paths

## Exam Traps
- Treating metrics and logs as interchangeable
- Configuring alerts without meaningful thresholds or action groups
- Assuming backup exists without checking policy and retention
- Skipping restore validation

## Practice Tasks
- Create a CPU alert for a VM
- Send diagnostics to a Log Analytics workspace
- Review logs for failed operations
- Configure backup for a VM and perform a test restore review
