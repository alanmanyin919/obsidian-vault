## Main Positioning Angle

- Be careful not to overclaim Python depth

## Section-By-Section Revision Plan

### 1. Summary Section
- Section to revise: `## Summary`
- Why: highlight enterprise software, databases, and web applications more clearly

**Before**
- `Software Engineer with 5+ years of experience delivering and supporting enterprise applications across frontend, backend, API integration, databases, and Azure-connected environments. Experienced in building business-critical workflows, troubleshooting production issues, improving deployment processes, and maintaining reliable application platforms. Strong background in cross-functional delivery, technical documentation, and secure system integration.`

**Revised version**
- `Software Engineer with 5+ years of experience developing and supporting enterprise web applications, backend services, APIs, and database-driven systems. Experienced in troubleshooting production issues, supporting business-critical software delivery, and maintaining reliable application platforms with strong technical documentation and problem-solving discipline.`

### 2. Technical Skills Section
- Section to revise: `## Technical Skills`
- Why: move JavaScript, SQL, and enterprise database work higher

**Before**
- `Languages: JavaScript, TypeScript, Java, Python`
- `Databases: Azure SQL Database, DB2, MongoDB, MariaDB, SQL Server-oriented query and schema work`

**Revised version**
- `Languages: JavaScript, Python, Java, TypeScript`
- `Databases: Azure SQL Database, DB2, MongoDB, MariaDB, SQL Server-oriented query and schema work`

### 3. What Not To Add
- Do not add `ERP`, `PLM`, `PHP`, or `ColdFusion`
- Do not claim `3+ years` professional Python experience unless that is fully supportable

## Suggested Edit Order
- Update the summary first
- Reorder the languages line second
