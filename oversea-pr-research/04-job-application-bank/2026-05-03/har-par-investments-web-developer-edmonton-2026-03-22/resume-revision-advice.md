## Section-By-Section Revision Plan

### 1. Summary Section
- Section to revise: `## Professional Summary`
- Why: the current summary is already better than before, but this posting wants web delivery, documentation, testing, and support work to read more directly.

**Before**
- `Software Developer with 5+ years of experience building and supporting web and enterprise applications across frontend development, backend services, API integration, and database-driven systems. Experienced in translating requirements into maintainable application features, supporting testing and troubleshooting, and improving reliability through practical documentation and structured problem-solving.`

**Revised version**
- `Software Developer with 5+ years of experience building and supporting web and enterprise applications across React-based frontend development, backend services, REST API integration, and database-driven systems. Experienced in translating requirements into maintainable website and application features, supporting testing and troubleshooting, and improving delivery quality through practical documentation and structured problem-solving.`

### 2. Contact / Header Section
- Section to revise: contact block
- Why: this posting asks directly about Canada work authorization and relocation.

**Before**
- `Location: Hong Kong`
- `Relocation: Open to relocation to Brampton`
- `Work authorization: Valid Canadian Open Work Permit | No sponsorship required`

**Revised version**
- Keep `Work authorization: Valid Canadian Open Work Permit | No sponsorship required`
- Change relocation line to something broader and safer such as:
- `Open to relocation within Canada`
- Reason: this posting is in Edmonton, not Brampton.

### 3. Technical Skills Section
- Section to revise: `## Technical Skills`
- Why: this posting explicitly names `HTML`, `CSS`, `JavaScript`, `TypeScript`, `React.js`, `JSON`, web service design, database management, and performance testing.

**Before**
- Skills are organized well, but web-targeted keywords are spread across sections.

**Revised version**
- Put the most relevant web stack nearer the top:
- `Frontend: React, HTML, CSS, JavaScript, TypeScript, Angular, Material UI, Redux Toolkit, React Native`
- `Backend: Spring Boot, NestJS, Express, FastAPI`
- `Databases: Azure SQL, MongoDB, DB2, MariaDB`
- `Practices: RESTful API design, system integration, RBAC, CI/CD, microservices, testing support`
- Add `JSON` only if you want it listed explicitly and you are comfortable supporting it.
- Do not add `ASP.NET`, `C#`, `Figma`, or `Adobe Illustrator` unless you can support them honestly.
