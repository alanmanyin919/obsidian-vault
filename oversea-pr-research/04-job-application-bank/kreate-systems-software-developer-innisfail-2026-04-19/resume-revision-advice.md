## Section-By-Section Revision Plan

### 1. Contact Section
- Section to revise: `## Contact`
- Why: the role is in person in Innisfail and location readiness matters

**Before**
- `Location: Relocating to Canada`
- `Work authorization: Canadian Open Work Permit, no sponsorship required`

**Revised version**
- `Location: Hong Kong | Open to relocation within Alberta`
- `Work authorization: Valid Canadian Open Work Permit | No sponsorship required`

### 2. Summary Section
- Section to revise: `## Summary`
- Why: the summary should stress web applications, APIs, databases, maintainability, and problem-solving rather than unsupported stack breadth

**Before**
- `Software Engineer with 5+ years of experience delivering and supporting enterprise applications across frontend, backend, API integration, databases, and Azure-connected environments. Experienced in building business-critical workflows, troubleshooting production issues, improving deployment processes, and maintaining reliable application platforms. Strong background in cross-functional delivery, technical documentation, and secure system integration.`

**Revised version**
- `Software Engineer with 5+ years of experience developing and supporting web and enterprise applications across frontend delivery, APIs, backend services, and database-driven systems. Experienced in troubleshooting production issues, improving maintainability, and supporting scalable application workflows with strong technical documentation and problem-solving discipline.`

### 3. Technical Skills Section
- Section to revise: `## Technical Skills`
- Why: keep JavaScript, APIs, databases, CI/CD, and Azure visible while avoiding unsupported stack claims

**Before**
- `Languages: TypeScript, JavaScript, Java, Python`
- `Backend And Integration: REST APIs, Spring Boot, NestJS, FastAPI, Microsoft Graph API`
- `Databases: Azure SQL Database, DB2, MongoDB, MariaDB, SQL`
- `Delivery Practices: troubleshooting, technical documentation, Git-based collaboration, CI/CD, iterative delivery`

**Revised version**
- `Languages: JavaScript, TypeScript, Java, Python`
- `Backend And Integration: REST APIs, Spring Boot, NestJS, FastAPI, Microsoft Graph API`
- `Databases: Azure SQL Database, DB2, MongoDB, MariaDB, SQL Server-oriented query and schema work`
- `Delivery Practices: Git-based collaboration, troubleshooting, technical documentation, CI/CD, iterative delivery`

### 4. Experience Sections
- Section to revise: `### System Developer` and `### Analyst Programmer`
- Why: lead with APIs, databases, JavaScript-based application work, debugging, and maintainability

**Before**
- `Build reusable React and Material UI components and data-driven views to improve maintainability and consistency across the platform.`
- `Design and integrate RESTful APIs for workflow execution, email processing, tagging, and role-based access control.`
- `Design and optimize Azure SQL queries, views, and functions for complex and performance-sensitive data retrieval.`

**Revised version**
- `Build and maintain web application features using React and reusable UI components to improve maintainability and responsive delivery.`
- `Design and integrate RESTful APIs to support workflow execution, application integrations, and backend communication.`
- `Design and optimize SQL queries, views, and functions to support complex data retrieval and performance-sensitive operations.`

### 5. What Not To Add
- Do not add `.NET`, `C#`, `Ruby on Rails`, `C++`, `VBA`, or strong Linux claims unless you can support them honestly
- Do not over-tailor to every keyword in this posting
