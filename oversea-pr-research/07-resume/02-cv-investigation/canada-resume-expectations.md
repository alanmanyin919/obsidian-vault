# Canada Resume Expectations

## Purpose
This note captures the working assumptions for a Canada-targeted resume so edits stay consistent.

## Core Direction
- Use a resume format, not a long CV format
- Keep it concise and recruiter-readable
- Prioritize relevance over completeness
- Emphasize outcomes, tools, and business impact where real evidence exists

## Likely Formatting Rules
- Prefer 1 page if experience is limited or not tightly relevant
- Use 2 pages only if the content is strong and still easy to scan
- Avoid photos, date of birth, marital status, HKID, or other personal details not needed for applications
- Use clear section headings and consistent bullet style
- Put location, contact details, LinkedIn, and portfolio or GitHub only if they help

## Content Rules
- Strong summary at the top only if it is specific, not generic
- Skills section should support the target role, not list everything ever touched
- Work experience bullets should focus on contribution, toolset, and outcome
- Projects should stay only if they prove practical ability or relevant domain fit
- Remove filler phrases, weak adjectives, and repeated wording

## Common Weaknesses To Watch For
- Bullets that describe duties without impact
- Too much emphasis on learning instead of delivering
- Technical terms without context
- Vague job titles that hide actual scope
- Large skill lists with no evidence in work history or projects

## Questions For Review
- Which role should the first resume target?
- Which past work is strongest for Canada employers?
- Which bullets can be rewritten with clearer outcomes?
- Which claims need proof, examples, or metrics?
