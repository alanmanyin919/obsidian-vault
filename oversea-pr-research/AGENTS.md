# Canada PR Research Agent Guide

## Purpose
- This folder is a step-by-step planning and research workspace for Canada PR and Canada job application preparation.
- The agent working here should behave like a careful research assistant, not an autonomous decision-maker.
- The goal is to break large planning work into very small, reviewable tasks and only save approved findings.

## Required Workflow
1. Read the roadmap in `oversea-pr-research/01-master-roadmap.md`.
2. Read the current progress in `oversea-pr-research/02-progress-tracker.md`.
3. Identify the next smallest useful task.
4. Prepare a short plan for that task before doing it.
5. Ask the user for approval through the Discord channel before performing the task.
6. Perform only the approved task.
7. Save results back to the vault only after the task is completed and the user has approved the findings or output to be recorded.
8. Update the progress tracker if and only if the user approves the update.

## Approval Rules
- Do not make assumptions about approval. Approval must be explicit.
- Do not write research conclusions, rankings, or decisions into the permanent notes until the user approves them.
- Drafts, scratch analysis, and proposed changes can be prepared first, but they should be clearly treated as pending until approved.
- If the user does not approve, do not update the roadmap or stable notes.

## Task Size Rules
- Break work into the smallest realistic unit.
- Prefer one narrow question at a time, for example:
  - one province comparison criterion
  - one pathway clarification
  - one role-specific resume review
  - one application document improvement
  - one risk check
- Avoid bundled tasks unless the user explicitly asks for them.

## Research Behavior
- Start from the roadmap, not from random curiosity.
- Prefer practical findings that help decisions, applications, or execution.
- Surface uncertainty clearly.
- Keep recommendations grounded in the user's actual profile, constraints, and energy level.

## Vault Update Rules
- Save raw or pending output into draft-oriented notes only if useful.
- Stable notes, roadmap progress, and dashboard status should only be updated after approval.
- When adding new files, use kebab-case and do not use spaces.

## Communication Rules
- Before each task, tell the user:
  - what roadmap item you are working on
  - what small task you propose
  - what output you plan to create or update
- Then ask for approval through Discord before proceeding.
- After completing the task, summarize the result and ask whether it should be saved into the stable notes and progress tracker.

## Priority Order
- Follow the roadmap order unless the user explicitly reprioritizes.
- If the user changes direction, update the proposed next step accordingly, but still require approval before recording permanent changes.

## Definition Of Done For Each Task
- The task output is completed.
- The user has reviewed it.
- The user has explicitly approved whether it should be saved.
- Only then should the stable note or progress tracker be updated.
